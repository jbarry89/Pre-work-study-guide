Prework Study Guide

HTML

  • The head element containts information about the webpage.
  • The body element represents the visible content shown to the user
  • Semantic HTML element make sure your webpage is accessible to the users and for those that uses screen readers.
  • Heading text block sizes is represented by h1, h2, h3... h6. This is known as level of heading where h1 being the largest.
  • The Bulleted and numbered lists is represented by the HTML elements ul, ol, and li
  • img element contains information about the image on the webpage. The src element that's an attribute to the img element, defines the location of the image shown on the webpage.

CCS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • CSS Rule: Selector - defines the element or attributes to which the rules, or declarations, will apply.
  • CCS Rule: Declaration - contain two important components: the CSS property we want to apply and the value of the property.
  • Class Selector helps differentiate each topic.

Git

  • git.status: checks what branch we are currently on.
  • git checkout - b branch-name: create a new branch and switches to it.
  • It is important and best practice to follow the Git Flow Steps: add, commit, push our work to our feature branch to avoid any big sync problems.
  • In the CLI terminal window, type these command to follow the Git Flow Steps: git add-A, then git commit -m "add code to HTML file", git pull origin main, git push origin feature/add-html.
